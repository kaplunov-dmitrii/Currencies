# 📊 Мониторинг курсов валют ЦБ РФ (WinForms)

Профессиональное десктопное приложение на платформе .NET Framework (C#), предназначенное для получения, анализа и конвертации актуальных и архивных данных о курсах валют, предоставляемых официальным API Центрального банка Российской Федерации.

## 🚀 Основной функционал

*   **Автоматическое обновление:** Асинхронная загрузка актуальных курсов валют на текущую дату сразу при старте приложения без блокировки интерфейса (UI).
*   **Архив данных:** Возможность выбора любой исторической даты через встроенный календарь `DateTimePicker` и загрузка архивных курсов валют.
*   **Умный конвертер валют:** Полноценный мультивалютный калькулятор, привязанный к выбранной архивной дате, с поддержкой отображения полных названий валют на русском языке.
*   **Экспорт отчетов (Ветка feat/export):** Дополнительный модуль для выгрузки результатов конвертации в текстовые файлы для последующего анализа.

## 🛠️ Архитектура и технологии

*   **Язык программирования:** C# (.NET Framework)
*   **Интерфейс:** Windows Forms (WinForms) с применением концепции кастомных стилей для таблиц (запрет редактирования, кастомные цвета выделения ячеек, отключение системной сортировки).
*   **Парсинг данных:** `System.Text.Json` (NuGet-пакет) для десериализации сложных древовидных JSON-структур ЦБ РФ в объектно-ориентированные модели C# через механизмы `Dictionary<string, T>`.
*   **Связывание данных (Data Binding):** Наполнение `ComboBox` живыми объектами классов данных с переопределением отображения через `DisplayMember` для повышения читаемости интерфейса.
*   **Безопасность (Валидация):** Бронебойная обработка ошибок пользователя с помощью конструкций `decimal.TryParse`, исключающая падение приложения при вводе некорректных данных или буквенных символов.

## 📈 Разработка и контроль версий (Git Workflow)

Проект разработан в строгом соответствии с методологией Git Flow:
1. Инициализирован локальный репозиторий, настроена фильтрация системного мусора сборки IDE с помощью `.gitignore`.
2. Реализована детальная история осмысленных коммитов, отражающая этапы проектирования моделей данных и интерфейса.
3. Отработаны навыки администрирования: исправление коммитов через `git commit --amend` и жесткие откаты состояний с помощью `git reset --hard`.
4. Интеграция новой функциональности (модуль экспорта) производилась в изолированной ветке `feat/export` с последующим слиянием в `main` через механизм **Pull Request** на сервере GitHub.
